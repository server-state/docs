
:::tip Templates

The `docs/_templates` folder contains templates for all commonly used page types
of the documentation.

All templates, aside from laying down a general structure that you have to fill
in, contain a few things to get you started:

1. Guidelines for when to use the template
1. Relevant component imports and copy/pastable snippets that might be useful
   for the topic
1. Writing tips for the type of content the topic is for

To use a template within VSCode, simply Ctrl+Drag it into the relevant folder,
rename it, and adjust it to your liking:

<Image src={require('./writers-guide/using-a-template.png').default} />

:::

## General Writing Guidelines

- Write in the present tense
- Use neutral pronouns (they/them instead of he/she and him/her)
- Be respectful to everyone
- Be aware of the potential for cultural misunderstandings

## Custom MDX Components

Apart from the elements Docusaurus natively provides, there are also a few
custom components for the Server State documentation:

### `<Grid />`

Allows to display several block elements next to each other, especially useful
in combination with [`<Reference />`](#reference-)

#### Code:

```mdx title='topic.mdx'
import { Grid } from '/components';

<Grid cols={2}>

:::danger First note

Note content

:::

:::caution Another Note

Note content

:::

</Grid>
```

#### Result:

<Grid cols={2}>

:::danger First note

Note content

:::

:::caution Another Note

Note content

:::

</Grid>

### Markdown content inside MDX components

To write Markdown content inside MDX components, leave a blank line before and
after your content and do not indent the content:

```mdx
<MyComponent>
    <MyChildComponent>

Some Markdown Content

    </MyChildComponent>

</MyComponent>
```

## Static Assets (images, videos and other files)

Static assets are stored within the `static` folder and served at the root of
the repository, i.e., an image in `static/img/logo.svg` can be referenced using
`/img/logo.svg`.

### Including an image

:::caution Big image sizes

Images with a big file size affect the page's load time and, through that, also
the search engine ranking. Therefore, when your image is bigger than 500 kB, try
to reduce its size with these steps before using them:

1. For diagrams and other "illustration-like" images, consider using a vector
   format (SVG)
1. Consider what size the image has to, reasonably, be displayed at, and reduce
   the image's dimensions if necessary.
1. Run your JPEG and PNG files through https://tinypng.com/ to reduce their file
   size without changing the image itself

:::

To include an image, you can use the `<Image />` component from above:

#### Code:

```mdx
<Image alt="Description of the image" src="/img/logo.svg" width="200" />
```

#### Result:

<Image alt="Description of the image" src="/img/logo.svg" width="200" />

### Images and dark mode

If your image has no problems with being inverted (light becomes dark and vice
versa), you can set the `invertible` attribute and allow the website to invert
them when users open the website in dark mode:

:::tip

One example of a well-supported image type is a diagram that only contains black
text/shapes on a white/transparent background.

:::

#### Code:

```mdx
<Image
	src="/static/img/drawio-diagrams/conventional-commits/conventional-commits-decision-table.drawio.svg"
	invertible
/>
```

#### Result:

<Image
	src="/static/img/drawio-diagrams/conventional-commits/conventional-commits-decision-table.drawio.svg"
	invertible
/>

(switch to dark mode to see the effect)
